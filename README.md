# TWODRAPES 订单工具

窗帘/帘布定制产品的订单管理、成本核算、利润计算和工厂反馈工具。数据保存到 SQLite，局域网内多台电脑共享产品、参数和订单。

## 安装与启动

```powershell
cd C:\Users\Administrator\Desktop\订单erp工具\订单工具
npm.cmd install
npm.cmd start
```

**要求：** Node.js 22+（内置 `node:sqlite`，无需安装 C++ 构建工具）。

启动后同时运行三个服务：

| 端口 | 用途 | 地址 |
|------|------|------|
| `/` | 统一登录入口 | `http://localhost:8080/` |
| `/admin` | 管理端 | `http://localhost:8080/admin` |
| `/ops/shopify` | 独立站运营端 | `http://localhost:8080/ops/shopify` |
| `/ops/amazon` | 亚马逊运营端 | `http://localhost:8080/ops/amazon` |

局域网访问：将 `localhost` 替换为服务器局域网 IP。

### 默认登录账号

系统默认要求登录，当前版本固定只保留 3 个系统账户：

- 管理端：`admin / admin`
- 独立站运营端：`twshop / tw123`
- 亚马逊运营端：`twama / tw123`

登录后界面左下角统一提供账户入口，支持：

- 查看当前登录用户信息
- 修改自己的显示名
- 修改自己的密码
- 退出登录

## 三端架构

### 统一入口与工作台路径

系统对外默认使用统一入口：根路径 `/` 为登录页，登录后会按权限自动进入 `/admin`、`/ops/shopify`、`/ops/amazon`。运营端共用同一套前端代码（`public/` 目录），通过当前路径识别工作台与渠道。**产品、订单、计算等数据按渠道完全隔离**，独立站与亚马逊互不串数据。

Amazon 端使用独立的品牌色主题（琥珀 + 深青），通过 CSS 变量 `[data-channel="amazon"]` 覆盖。

**认证行为：**
- 无 token 或 token 失效时自动回到 `/login`
- token 渠道与当前端口不匹配时拒绝进入
- 管理员登录后默认进入 `/admin`，并可从账户入口直接跳转查看两个运营端
- 普通用户只能进入自己的运营端，且不提供切换到管理端的入口

**功能页面：**
- **下单** — 客户信息、产品选择、实时价格预览、剪贴板批量解析、折扣计算
- **订单** — 订单列表（支持展开查看项目颜色/面料/尺寸）、编辑、导出、CSV 导入、批量操作；支持日期范围筛选、周期快速选择（本月/上月/近3月/近6月）、筛选标签（filter chips）
- **产品控制** — 产品 CRUD、价格表、选项组编辑、Easify CSV 导入、按渠道归档
- **拼接计算器** — 判断是否需要拼接、用料估算、裁剪示意图
- **利润计算** — 单订单利润明细、金额修正、工厂成本反馈导入
- **数据分析** — 按渠道独立统计收入/支出/利润、月趋势图、支出结构饼图、产品对比表
- **提醒系统** — 订单提醒设置、通知面板（分组显示、点击跳转、一键清除）

**工作台识别原理：** 前端优先通过路径识别当前工作台与渠道，并兼容旧端口访问。修改 `public/` 中的任意文件，两个运营端路径同步生效。

### 管理端（兼容旧端口 8081）

面向管理层的汇总视图。对外正式入口为 `/admin`，`8081` 仅作为兼容保留。可查看所有渠道订单、利润详情、导入工厂成本反馈、控制全局公式参数。

**认证行为：**
- 仅管理员可进入管理端业务页
- 管理员可继续通过左下角账户入口进入两个运营端
- “用户管理”当前仅用于查看固定系统账户；个人资料与密码修改走各端账户入口

**功能页面：**
- **订单汇总** — 全渠道订单列表，支持日期范围筛选、周期快速选择、渠道/状态/搜索筛选，筛选标签（filter chips），统计卡片，点击订单号查看订单详情（项目按产品分组展示），提醒管理
- **数据分析** — 按渠道独立统计收入/支出/利润、月趋势图、支出结构饼图、产品对比表，支持日期范围和产品筛选
- **参数设置** — 全局参数、面料/内衬、加工费/定型费规则的增删改查
- **用户管理** — 固定系统账户查看页，按账户归属筛选

## 可选环境变量

项目不依赖 `.env` 文件即可运行；如需固定本机配置，可参考 `.env.example`：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `8080` | 主入口端口 |
| `FACTORY_PORT` | `8081` | 管理端兼容端口 |
| `AMAZON_PORT` | `8082` | 亚马逊端兼容端口 |
| `HOST` | `0.0.0.0` | 监听地址 |
| `PAYPAL_FEE_RATE` | `0.044` | PayPal 手续费率 |
| `USD_RMB_RATE` | `6.9` | USD/RMB 汇率 |
| `NO_AUTH` | — | 设为 `1` 或 `true` 显式关闭认证；默认要求登录 |
| `ADMIN_USER` | `admin` | 管理员用户名 |
| `ADMIN_PASSWORD` | `admin` | 管理员密码 |
| `MULTI_PORT` | — | 设为 `1` 或 `true` 时在本地继续监听 `8081/8082` 兼容端口；生产默认单端口 |
| `NO_AUTO_LISTEN` | — | 设为 `1` 不自动监听（用于测试） |
| `TEST_DB` | — | 测试数据库路径 |

说明：

- 默认情况下，不配置 `NO_AUTH` 也会启用登录
- 只有在本地排障时才建议设置 `NO_AUTH=1`
- `.env` 中的 `ADMIN_USER` / `ADMIN_PASSWORD` 属于兼容保留配置，不是默认业务登录账号

## 数据位置

| 路径 | 说明 |
|------|------|
| `./data/twodrapes.sqlite` | SQLite 数据库 |
| `./data/twodrapes.sqlite-shm` / `-wal` | WAL 日志文件 |
| `./data/backups` | 自动/手动备份 |
| `./data/exports` | 导出目录 |
| `./data/delivery-screenshots` | 发货截图 |
| `./data/.jwt-secret` | JWT 签名密钥（自动生成） |
| `./data/.admin-password` | 管理员密码哈希（自动生成） |

首次启动自动初始化数据库，运行 26 个迁移脚本，并从种子数据提取现有产品、价格表和选项。

本项目使用 Node.js 22+ 自带的 `node:sqlite`（`DatabaseSync`，同步 API），不需要安装 Visual Studio C++ 构建工具。启动时看到 SQLite experimental warning 属于 Node 当前提示，不影响使用。

## 运营端功能

### 下订单

1. 填写订单信息（订单号、客户姓名/邮箱/电话、收货地址）
2. 选择下单日期，交期默认为空（后续填入）
3. 选择产品、输入尺寸（Width/Length inch）、数量、褶皱倍率
4. 选择产品选项（颜色、挂钩工艺、内衬等），实时预览价格和折扣
5. 点击"添加到当前订单"，支持继续录入（保留选项，仅清除尺寸）
6. 一个订单可包含多个项目，保存订单到数据库

**剪贴板解析：** 点击"信息匹配"，粘贴订单信息，自动识别并填充客户字段（订单号、姓名、邮箱、电话、地址）。

### 订单

- 表格展示当前渠道订单，项目列按产品分组紧凑展示
- 订单状态：草稿 → 待生产 → 待发货 → 完成
- 支持日期范围筛选、周期快速选择（本月/上月/近3月/近6月/全部）、搜索、状态筛选、产品筛选、生产成本/物流成本录入状态筛选
- 筛选标签（filter chips）：当前激活的筛选条件以标签形式显示，可逐个清除或一键清除
- 批量操作：批量导出 CSV、批量重新计算、批量删除
- 点击订单号打开详情弹窗，查看完整订单信息和项目明细
- 编辑模式可修改客户信息、物流信息、项目数量、选项值（所有状态均可编辑）
- 物流信息：货代、尾程派送渠道、尾程追踪编码、送达日期、重量 KG、物流成本
- 发货截图上传/查看

### 产品控制

- 产品选择、保存、复制、删除
- **按渠道隔离**：独立站端和亚马逊端各自维护独立的产品列表
- 产品归档：按渠道独立归档，归档产品可在"归档产品"列表中查看和恢复
- 基础信息：名称、默认主面料、基础价、每单位片数（`panels_per_unit`）
- Width/Length 价格表：可新增/删除价格档位
- 产品选项组：支持拖拽排序，每个选项组可配置多个选项值（标签/售价USD/成本RMB）
- 导入：Easify CSV 导入、产品 CSV 导入、下载模板

### 拼接计算器

- 订单批量计算：选择订单后勾选项目，统一计算是否需要拼接和用料
- 单个试算：临时输入尺寸做试算
- 判断逻辑：宽方向和高方向需用量 vs 门幅，超过门幅则需拼接

### 利润计算

- 选择订单加载利润明细（销售额、税费、收入、物流成本、总成本、利润率）
- 调整参数重算：物流成本、销售 USD 覆盖、税费 USD 覆盖
- 项目利润明细表
- 成本诊断：系统预计 vs 工厂成本 vs 最终采用成本对比
- 导出加工费用确认单 XLSX

### 数据分析

- 按渠道独立切换查看（独立站 / 亚马逊）
- 汇总卡片：总收入、总支出、总利润、利润率、订单数
- 收支趋势图：月度收入/支出/利润/样品折线图
- 支出结构图：面料/内衬/加工/拼接/定型/选项/物流占比
- 产品对比表：按收入/利润/利润率/销量排序
- 日期范围和产品筛选

### 提醒系统

- 订单级别提醒设置（提醒开关 + 提醒文本）
- 通知面板：铃铛图标 + 未读计数徽章（脉冲动画）
- 分组显示：待处理提醒、近期完成订单
- 点击跳转到订单详情
- 一键清除提醒

## 订单金额口径

| 字段 | 说明 |
|------|------|
| 实际销售 USD | 销售 USD（不含税）+ 税费 USD |
| 实际销售 RMB | 实际销售 USD × 汇率（globals 表 `usdRmbRate`，默认 6.9） |
| 扣手续费后收入 RMB | 实际销售 USD × (1 - 手续费率) × 汇率 |
| 利润 RMB | 扣手续费后收入 RMB - 最终成本 RMB |
| 最终成本 RMB | 订单项目最终采用成本合计 + 物流成本 |

## 成本计算口径

系统下单时自动计算"系统预计成本"。工厂反馈导入后，按优先级生成"最终采用成本"：

1. 有"结算" → 使用工厂结算
2. 无结算但有"工厂成本合计" → 使用工厂成本合计
3. 无工厂反馈 → 使用系统预计成本

**预计成本拆分：** 主面料成本、内衬成本、加工费、拼接费、定型费、选项成本、物流成本、成本合计。

**拼接判断逻辑：** 计算宽方向需用量（成品宽 × 褶皱倍率 + 余量）和高方向需用量（成品高 + 余量）。仅当两个方向都超过门幅时才需要拼接，任一方向可整片裁剪则无需拼接。

**内衬成本：** 工厂反馈支持独立的内衬用料和成本录入（内衬材料、实际用料、单价、加工费、定型费、成本合计、结算），与主面料成本分开计算后合并。

**默认参数：**

| 参数 | 默认值 | 用途 |
|------|--------|------|
| 上包衬余量 | 10 cm | 面料理论用料 |
| 底边余量 | 5 cm | 面料理论用料 |
| 总余量 | 24 cm（单层）/ 26 cm（双层） | 宽方向和高方向的固定余量 |
| 发料安全余量 | 50 cm | 仅发料成本，不影响加工/定型/拼接 |

高度分档使用下单成品高度，加工费按计费宽度计算，拼接费单独列出。订单详情显示"成本诊断"，对比系统预计、工厂成本、最终采用成本及差异。

## 工厂反馈流程

1. 运营端或管理端在"利润计算"页导出加工费用确认单 XLSX
2. 工厂填写实际用料、面料单价、加工费、定型费、成本合计、结算、完成时间、备注（含内衬明细）
3. 运营端或管理端导入填写后的 XLSX，系统按"品名/编号"自动匹配订单项目
4. 系统按优先级确定最终成本（结算 > 成本合计 > 系统预计）

## 注意事项

- `localStorage` 只用于当前订单草稿，不是正式数据源
- 正式订单、产品、参数都写入服务器 SQLite
- **渠道数据完全隔离**：产品、订单、归档、计算等数据按渠道独立存储。独立站端创建的产品不会出现在亚马逊端，反之亦然
- 面料、内衬、全局参数、加工费/定型费规则为全渠道共享（工厂级设置）
- 税率为参考平均值，实际以客户地址/ZIP 为准
- 销售汇率和 PayPal 手续费通过 globals 表运行时读取（管理端修改后即时生效，默认 6.9 / 4.4%）
- 管理端对外正式入口为 `/admin`；兼容旧端口 `8081` 仅用于内部排障或历史访问
- 运营端对外正式入口为 `/ops/shopify` 和 `/ops/amazon`；兼容旧端口 `8080/8082` 可继续保留
- 如修改了 `src/routes/auth.js`、`src/config.js` 或 `.env`，需要重启服务后认证行为才会生效
- CORS 仅允许 localhost 来源，外部网站无法跨域访问 API
- 部署到新服务器时，`data/` 目录下的数据库和备份需一起迁移；认证密钥（`.jwt-secret`、`.admin-password`）会自动生成新的
- 部署到 Hostinger 时，推荐直接导入 GitHub 仓库，并在面板中单独配置环境变量、单独上传 `data/twodrapes.sqlite`
